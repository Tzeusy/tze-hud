# 2026-05-11 Goals: External Agent Projection Authority

## Objective

Design and implement a v1-scoped external agent/projection authority that can launch or attach provider-neutral LLM sessions, authenticate them to the local Windows `tze_hud` runtime, and route each session to bounded HUD presence through existing v1 surfaces: zones, widgets, and leased text-stream/raw-tile portals.

The authority must remain outside the compositor. The runtime remains sovereign over pixels, timing, composition, attention policy, privacy/redaction, capability grants, lease TTLs, revocation, safe mode, and resource budgets.

## Scope Guardrails

- Windows is the active deployment target.
- No WebRTC, media, mobile, glasses, or embodied presence.
- No terminal capture, PTY injection, or raw keystroke passthrough.
- No LLM in the frame loop.
- No agent-rendered chrome or new compositor-owned UI primitive.
- No notification-spam model; attention requests are classified, rate-limited, dismissible, and ambient by default.

## Design Direction

The existing `cooperative-hud-projection` capability already supplies the low-token operation contract, memory-only projection state, owner-token authorization, semantic HUD input inbox, and resident gRPC text-stream portal adapter. The smallest missing contract for this goal is the layer above it: an external session authority that can register launched or attached provider-neutral LLM sessions and map each session's presence intent to one governed runtime surface.

The first vertical slice will stay in `tze_hud_projection` and model:

- provider-neutral launched vs attached sessions;
- Windows HUD target metadata without persisting secrets;
- governed surface routing to zone, widget, or text-stream portal paths;
- multi-session lifecycle state and bounded cleanup/reconnect bookkeeping;
- a deterministic three-session demo plan that proves one session per surface class.
